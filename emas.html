Motivation

* Gap: academic agent programming frameworks & engineering reality * Lack of intuitive, practical entry point for non-experts * Outdated technology stacks in MAS research * Reference point: *Machine Learning* community
Mascardi et al. "Engineering Multi-Agent Systems: State of Affairs and the Road Ahead." Logan. "An agent programming manifesto."

* Usability over intellectual elegance * Flexibility over rigor * Extensibility over out-of-the-box power

Example code I

```JavaScript const beliefs = { ...Belief('door', { locked: true }), ...Belief('requests', []) } ``` *Comprehensive examples + docmentation: [https://github.com/TimKam/JS-son](https://github.com/TimKam/JS-son)*

Example code II

```JavaScript const plansPorter = [ Plan( beliefs => !beliefs.door.locked && beliefs.requests.includes('lock'), () => [{ door: 'lock' }] ), Plan( beliefs => beliefs.door.locked && beliefs.requests.includes('unlock'), () => [{ door: 'unlock' }] ) ] ```

Example code III

```JavaScript const porter = new Agent( 'porter', beliefs, {}, plansPorter ) ```
